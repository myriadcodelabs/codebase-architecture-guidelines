# Contributing

Thanks for contributing.

## Policy

- Pull requests are optional and not required.
- If you adapt any guideline file, open an issue in this repository and link your adapted version.
- Keep attribution to this repository in downstream copies.
- Include a link to your adapted version so others can discover improvements.

## What To Include In The Issue

1. Which file(s) you adapted.
2. A short summary of what changed and why.
3. A link to your adapted version (repository, gist, or docs URL).

## Optional Pull Request

If you want your changes considered upstream, open a PR with:

1. A clear rationale for the change.
2. The intended impact on directory structure or architecture guidance.
3. Any compatibility notes for existing adopters.
